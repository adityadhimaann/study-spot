📅 Date: May 2026 👤 Developer: Aditya Dhiman 🏷️ Type: Working Website 📦 Version: 1.0

1. Project Overview

StudySpace is a comprehensive, full-stack library management platform designed to streamline how students discover, reserve, and utilize academic study environments. It addresses the critical problem of inefficient space management in university libraries by providing a digital-first booking experience with real-time availability, interactive floor maps, and data-driven administrative tools.

The platform serves two distinct user roles: Students who can browse rooms, make bookings, and manage their reservations; and Administrators who can monitor space utilization, manage rooms, approve/reject bookings, and respond to student feedback.

Key Value Proposition: StudySpace reduces the average time students spend finding a study spot from ~20 minutes to under 30 seconds, while giving librarians full visibility into space utilization patterns.

2. Project Description (Brief)

StudySpace is a production-ready web application built with React 19 and Express.js that digitizes university library space management. It features a 4-step booking wizard, interactive SVG-based floor maps with multi-floor navigation, real-time session tracking with countdown timers, and a comprehensive admin dashboard with analytics (room usage charts, peak hour visualization, student directory). The platform uses JWT-based dual authentication (student/admin), MongoDB Atlas for cloud data storage, and Cloudinary for media management. Key highlights include animated UI transitions via Framer Motion, responsive design with Tailwind CSS 4, and role-based access control across 7 RESTful API route modules.

5. System Architecture

The application follows a client-server architecture with clear separation of concerns:

  • Frontend (SPA): React 19 single-page application with file-based routing via TanStack Router. All pages are client-rendered with data fetched from the backend API.
  • Backend (REST API): Express.js server exposing 7 route modules: /api/auth, /api/rooms, /api/bookings, /api/admin, /api/notifications, /api/feedback, /api/storage.
  • Database: MongoDB Atlas cluster with collections for Users, Rooms, Bookings, Feedback, and Notifications. Mongoose ODM for schema validation and query building.
  • Authentication: JWT-based stateless auth with role-based middleware guards. Admin routes protected by isAdmin middleware. Frontend route guards via TanStack Router's beforeLoad.
  • Media Pipeline: Multer for file upload handling → Cloudinary SDK for cloud storage and CDN delivery.

8. Database Schema

User Model

Fields: name, email (unique), password (hashed), role (student/admin), department, year, studentId (unique), profileImage

Room Model

Fields: name, type (quiet/group), capacity, status (available/almost-full/booked), floor, amenities[], rating, mapCoords (x, y, w, h)

Booking Model

Fields: user (ref), room (ref), date, slot, status (upcoming/completed/cancelled), timestamps

Feedback Model

Fields: user (ref), message, type (bug/feature/general), status (new/resolved), timestamps

9. How to Run Locally

  1. Clone: git clone https://github.com/adityadhimaann/study-spot.git
  2. Install frontend: npm install (from root directory)
  3. Install backend: cd backend && npm install
  4. Configure: Create backend/.env with MONGO_URI, JWT_SECRET, Cloudinary keys
  5. Seed data: node seed.js && node adminSeed.js && node studentSeed.js
  6. Start backend: npm run dev (from backend/)
  7. Start frontend: npm run dev (from root/)
  8. Access: https://main.dgxs91r156db5.amplifyapp.com/
