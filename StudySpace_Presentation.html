🎯 The Problem

Why StudySpace?

University libraries face chronic space management challenges:

  • Students waste 15-20 minutes searching for available study spots
  • No centralized system for room reservation or availability tracking
  • Librarians lack tools to monitor space utilization and peak hours
  • Double-bookings and overcrowding during exam periods
  • No feedback channel between students and library administration

⚙️ Technology

Tech Stack & Architecture

Frontend

⚛️ React 19 🧭 TanStack Router 🎨 Tailwind CSS 4 ✨ Framer Motion 📐 Radix UI ⚡ Vite 7 📝 TypeScript 📊 Recharts

Backend

🟢 Node.js 🚂 Express.js 5 🍃 MongoDB Atlas 🔑 JWT Auth ☁️ Cloudinary 🔒 bcrypt.js

Architecture

  • SPA frontend with file-based routing (TanStack)
  • RESTful API backend with 7 route modules
  • MongoDB Atlas cloud database with Mongoose ODM
  • JWT token-based auth with role guards
  • Cloudinary CDN for image/media hosting
  • AWS Amplify (frontend) + Elastic Beanstalk (backend)

🔍 Features Deep Dive

Key Differentiators

🗺️ Interactive SVG Floor Map

  • Multi-floor navigation (1st, 2nd, 3rd floor)
  • Color-coded room status (available/booked)
  • Click-to-book directly from map
  • Admin map editor for coordinates

📊 Admin Analytics Dashboard

  • Real-time booking statistics & trends
  • Room usage percentage bars
  • Peak hour heat-map visualization
  • Student directory with search filters

⏱️ Live Session Tracker

  • Active booking detection on dashboard
  • Real-time countdown timer display
  • Session status badges (Active/Completed)
  • Auto-detect current time slot

💬 Feedback & Notifications

  • In-app feedback widget (bug/feature/general)
  • Admin notification bell with live count
  • Approve/reject booking requests inline
  • Resolve/reopen feedback tickets
